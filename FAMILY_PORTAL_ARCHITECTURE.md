# Family Portal Architecture

## Purpose

Give families continuity after the first conversation without forcing accounts before value is delivered.

## Future Modules

- Identity layer
- Consent and privacy settings
- Guidance history
- Report archive
- Action plans
- Progress tracking
- Follow-up reminders
- Data export
- Deletion controls
- Subscription readiness

## Memory Layer

Future memory should store user-approved case continuity, not hidden profiling.

## Build Later

Do not build until repeated users ask to return, save progress, or manage follow-up over time.
