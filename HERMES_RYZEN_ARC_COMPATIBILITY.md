# Hermes / Ryzen / ARC Compatibility

## Current Compatibility

The narrative layer remains inside the existing service boundary:

- `contextExtraction.ts`
- `conversationIntelligence.ts`
- `staticGuidance.ts`

Future Hermes can replace deterministic generation while keeping persistence, safety, follow-up, and founder alerts stable.

## Future Hermes Readiness

Hermes could later read:

- narrative memory
- pressure source patterns
- underlying needs
- central human questions
- validated reflection strategies
- guidance outcomes

## Future ARC Readiness

FamilieKompas can evolve into a Family Support ARC by preserving:

- family-support heuristics
- validated guidance patterns
- support navigation methods
- safety and escalation lessons
- founder-reviewed outcome patterns

## Future Ryzen Readiness

Ryzen could later receive aggregated, non-personal intelligence:

- narrative patterns
- support gap patterns
- guidance strategy maturity
- ARC specialization signals

No raw conversations, personal profiles, or identifiable family situations should flow upward.

## Not Implemented

- Hermes integration
- Ryzen integration
- ARC infrastructure
- memory federation
- knowledge synchronization

## Phase 6 Convergence Boundary

`HERMES_INTEGRATION_PATH.md` is the controlling Hermes implementation path. `PHASE_6_CONVERGENCE_ROADMAP.md` is the controlling sequence. This document remains a compatibility note and should not be used to justify implementing Hermes, Ryzen, ARC infrastructure, or knowledge synchronization during Phase 6.
