# Hermes / Ryzen Knowledge Continuity Readiness

## Current Readiness Assessment

FamilieKompas is now structured around a replaceable conversation intelligence boundary:

- `lib/conversation/contextExtraction.ts`
- `lib/conversation/conversationIntelligence.ts`
- `lib/guidance/staticGuidance.ts`

This is suitable for future Hermes integration because the product UI and persistence layer do not need to know whether guidance is deterministic or generated by a future intelligence layer.

Current readiness strengths:

- message-based conversations
- structured context extraction
- deterministic pattern labels
- guidance result generation behind a service boundary
- safety logic separated from guidance tone
- founder alert logic preserved

Current readiness limits:

- no knowledge archive exists
- no skill evaluation loop exists
- no cross-case learning exists
- no user-consented knowledge extraction pipeline exists

## Future Knowledge Archive Opportunities

Future versions could preserve non-personal ARC knowledge such as:

- recurring parent situations
- recurring emotional patterns
- recurring support gaps
- recurring founder-attention patterns
- effective reflection questions
- effective guidance structures
- support recommendations that are validated by human review
- guidance approaches that users mark as unclear or unhelpful

This should be stored as generalized domain knowledge, not personal case memory.

## Future Skill Preservation Opportunities

FamilieKompas could later preserve:

- validated listening strategies
- successful conversation pacing patterns
- reflection questions that lead to clarity
- support-navigation methods that reduce overwhelm
- domain maturity indicators such as safety accuracy, helpfulness ratings, and founder validation

These should become ARC-level guidance capabilities, not user profiles.

## Future Ryzen Handoff Opportunities

A future path could be:

FamilieKompas ARC -> Hermes Knowledge Archive -> Ryzen Knowledge Convergence Layer

The handoff should contain:

- generalized guidance patterns
- validated methods
- anonymized support-gap insights
- domain capability improvements
- safety and boundary learnings

It should not contain:

- raw conversations
- personal identifiers
- identifiable family situations
- unreviewed sensitive case material

## Architectural Recommendations

- Keep `conversationIntelligence.ts` as the main replacement boundary.
- Keep safety detection separate from empathic guidance generation.
- Add future knowledge capture only after consent, governance, anonymization, and founder review are designed.
- Treat useful guidance patterns as versioned methodology assets.
- Separate product analytics from ARC knowledge preservation.

## Outside MVP Scope

The following remain outside the MVP:

- Hermes integration
- Ryzen integration
- memory federation
- knowledge synchronization
- retention systems
- account-based learning
- automated cross-case training
- personal user memory
