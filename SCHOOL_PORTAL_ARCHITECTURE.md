# School Portal Architecture

## Purpose

Support school-family collaboration without turning FamilieKompas into school administration software.

## Future Capabilities

- Referral requests
- Parent collaboration summaries
- Guidance report sharing with consent
- Safeguarding workflow
- Teacher support layer
- Communication boundaries

## Boundaries

Schools should not access family data without explicit parent consent. Safeguarding flows must remain conservative.

## Convergence Boundary

School tooling is later-stage infrastructure. It should not be built before family value, founder workflow, consent patterns, and manual school collaboration needs are validated.
